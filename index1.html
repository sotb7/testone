ELEMENTO

Los Elementos conforman el lenguaje HTML
Están formados por una etiqueta de apertura y una etiqueta de cierre. Ejemplo: < h1 > contenido < /h1 >

Algunos elementos no tienen contenido por lo tanto están conformados por sólo una etiqueta. Ejemplo: < br > es el elemento line break (realiza un salto de renglón)


Lo primero que se escribe al comenzar un archivo HTML es el doctype que es también llamado declaración de tipo de documento. Le dice a los navegadores con que versión de HTML vamos a trabajar.
El doctype es obligatorio y fundamental, para que la interpretación de las etiquetas que trabajamos dentro de nuestro documento sea correcta.
El doctype en HTML 5 es:

< !doctype html > como las etiquetas en HTML5 no son case sensitive, se podría escribir Doctype o DOCTYPE y funcionaría igualmente

Después de la declaración del doctype, van los siguientes elementos:

< html >

< head > anidados entre apertura y cierre de la etiqueta html colocamos un segundo elemento llamado head. En el head va la información que brindamos a los motores de búsqueda y al navegador, como también las vinculaciones a CSS y/o Javascript

< title > anidado dentro del elemento head va el elemento title que muestra aquello que va en el tab o pestaña del navegador

< /title >

< /head >

< body > es el elemento que va después del head y es donde vamos a colocar todos los elementos que son visibles de nuestro sitio web. Texto, immágenes, tablas

< /body >

< /html >

Se puede prescindir de escribir elementos en nuestro archivo HTML, como por ejemplo body y head, y escribir sólo title por ejemplo, es váido y el navegador interpreta el HTML sin tener todos los elementos, pero no es recomendable (porque más adelante cuando incluyamos librerías de javascript para poder hacer que ciertos elementos sean visibles en versiones antiguas por ejemplo de internet explorer, si no tenemos esos elementos las librerías no van a funcionar)

Atributos y Valores 2

Meta

Provee información adicional para el documento. Ya sea para el propio navegador como para el robot de búsqueda.

El contenido de "meta" no es visible desde el navegador.

Entre los más importantes se encuentran el meta charset y el meta description (en otro momento fueron muy usados meta keywords o meta autor).

También hay meta como el viewport relacionados con el diseño responsivo, y la adaptabilidad de la página al ancho de diferentes dispositivos que uso para ver la página.

En el meta description se ponen las cosas que van a aparecer en el buscador debajo del nombre de nuestra página

< meta charset="iso-8859-1!" > El iso-8859-1 es el que contiene los caractéres de los idiomas Español, Portugués y Francés

Enunciados o Encabezados

Los enunciados tienen el elemento "h" (header) y van del h1 al h6 (va decreciendo el valor de importancia).

No se usan por el styling, sino por la semántica, ya que los robots de búsqueda les van a prestar atención en orden de h1 a h6.

Los títulos van a tener diferente importancia de acuerdo a la sección, pero los elementos "h" no se utilizan para subtítulos (la W3C no lo recomienda por el valor semámtico)

Nada dentro del HTML debe ir sin formato, hay que colocarlos siempre dentro de un elemento (ejemplos: p que es el paragraph, h header). No es necesario usar todos los niveles de h

UTF8 with BOM: cuando no detectó caractéres y en la página salió cualquier cosa, en el SUBLIME TEXT va a FILE -- SAVE WITH ENCODING -- UTF8 with BOM (en el Visual Studio Code no se como sería esto porque no figura Save with encoding)

ELEMENTOS DE FORMATO

Elementos de Línea y de Bloque

Sirven para destacar cierto contenido pero a la vez tienen un fin semántico. Y se pueden anidar dentro de otros elementos como p o h por ejemplo, debido a que son elementos de línea, y por este motivo van anidados dentro de elementos de bloque y no puede darse al reves (un elemento de bloque no va anidado dentro de un elemento de línea como strong o los demás)

Ejemplos: etiqueta < strong > DESTACA EL CONTENIDO ESTE; LO RESALTA COMO NEGRITA PERO TIENE UN FIN SEMANTICO A DIFERENCIA DE BOLD < /strong>

Otro ejemplo es el elemento de formato < em > (emphasis) Que hace cursiva lo que está acá adentro de estas etiquetas; pero también tiene un fin semántico < /em >

Hay otros elementos que tienen este tipo de styling, como son las etiquetas < b > que harían lo de acá adentro bold < /b > y la < i > y lo de acá adentro italic < /i > pero la diferencia con strong y em es que b e i no tienen un fin semántico

Los elementos de línea, conviven en la misma línea que otros elementos sin ningún problema, en cambio los elementos de bloque siempre se ubican uno debajo de otro

Los elementos de bloque son los párrafos y los enunciados

Los elemntos de línea son los elementos strong, em, b e i

Tampoco puede anidarse un párrafo dentro de otro párrafo ni un enunciado dentro de otro enunciado, ni un párrafo dentro de un enunciado, ni un enunciado dentro de un párrafo

Hay excepcciones a la regla en las que veremos más adelante, en los que hay ciertos elementos que estructuralemnte tienen a su vez otros elementos de bloque anidados siendo los elementos padre (los elementos contenedores) también de bloque

CSS (Cascading Style Sheet)

Es una hoja de estilo en cascada, la definición viene dada de como este lenguaje va dando estilo a los elementos del HTML. Es por lo tanto un complemento de HTML, que va a decir como se van a visualizar los elementos que creamos en HTML

CSS trabaja de forma interna, externa como en línea; es decir que hay 3 formas de implementar CSS en HTML

CSS EXTERNO

El CSS externo puede utilizarse para afectar a todo el sitio web

Se hace a través de un archivo externo de CSS y se trabaja a través de un elemento link, el cual tiene el atributo rel cuyo valor es "stylesheet" el atributo type cuyo valor es "text/css" y el atributo href cuyo valor es la ruta donde está guardado el archivo CSS "theme.css"(en el ejemplo que vemos que es < link rel="stylesheet" type="text/css" href="theme.css" >

El elemento link siempre va en el head del HTML

En HTML5 se puede omitir el atributo type



CSS INTERNO

Tiene las etiquetas style dentro del head

El CSS interno afecta sólo al archivo en el cual me encuentro (en el que escribimos las instrucciones CSS)

CSS INLINE o EN LINEA

Se usa en la línea directamente con el atributo style que tiene una propiedad y un valor

Sólo afecta a la etiqueta en la cual se encuentra



FUENTES

Las fuentes seguras o de sistema, son las que vienen predeterminadas en los sistemas operativos más populares, mientras que las fuentes de fantasía se pueden trabajar en CSS3 con la regla add font face. Pero inicialmente debemos contar con que el usuario tenga la fuente en su computadora. Una fuente segura es Arial. Es indistinto que esté escrita en mayúsculas o minúsculas. Igual es recomendable hacer un camino de fuentes.

Lo siguiente es un camino de fuentes, primero trata de usar la primer tipografía, luego la siguiente y la familia que puse al final como sans-serif "font-family: arial, 'Arial black', sans-serif" prestar atención que los nombres de tipografías compuestos se escriben con comillas simples

Si trabajamos sobre un elemento contenedor, como el body, que contiene todo lo que es visible en la página web, vamos a notar que si modificamos la tipografía la va a cambiar en todo el sitio web. Salvo en algún elemento que tenga indicada otra tipografía, como por ejemplo h1 style=font-family: verdana;> /h1 >

En CSS tenemos lo que se llama Ley de Herencia. En vez de escribir múltiples veces la tipografía, la escribo en un elemento contenedor y los elementos que están dentro la van a heredar. Pero si a uno de esos elementos, que potencialmente heredarían la letra que fijé en el elemento contenedor, les hago una modificación inline y esto sobre escribiría el valor heredado y va a adquirir el valor modificado
